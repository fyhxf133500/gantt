# Changelog

记录项目的重要变更。

## [2026-04-28]
### Added
- 甘特图任务条支持双击编辑：复用现有任务编辑弹窗，普通任务、里程碑和父任务均可从甘特图直接打开编辑，保留原有字段限制与交互逻辑。

## [2026-04-27]
### Added
- 里程碑新增节点状态字段：`milestoneStatus` 与 `passedAt`，支持未确认、待确认、已延期、已通过状态展示。
- 任务列表新增“状态”列，里程碑可在列表中手动“确认通过”或“撤销通过”。
- 新增通用派生计划状态 `scheduleStatus`，普通任务和父任务可展示未开始、进行中、已完成、已延期。
- 甘特图主页面新增项目健康概览条，展示总任务、进行中、已完成、已延期、待确认节点、已通过节点和全局关键任务统计。
- 项目健康概览条支持点击已延期、待确认节点、已完成、全局关键任务，自动切换到对应任务筛选。
- 新增依赖状态校验，任务进度与 FS/SS/FF 依赖关系冲突时在状态列显示“依赖未满足 / 依赖异常”。
- 依赖异常 UI 改为小型红色告警图标，点击后通过轻量浮层查看当前任务、前置任务、依赖类型和冲突说明。
- milestone 依赖未满足时的强制通过确认从浏览器原生弹窗改为项目内居中自定义弹窗，位置和样式更稳定。
- “状态”表头新增单个问号帮助图标，点击后可查看未开始、进行中、已完成、已延期、待确认、已通过和红色告警图标含义。
- 新增项目模板能力，项目支持 `isTemplate` 标记并与普通项目一起持久化到 `projects` localStorage。
- 项目更多菜单新增“保存为模板”，会复制当前项目任务、父子层级和依赖关系，并重新生成项目 id 与任务 id。
- Sidebar 新增“模板”分组，普通项目和模板分组展示，项目搜索会同时搜索两个分组。
- 模板菜单新增“从模板创建项目”，会生成普通项目并自动切换到新项目。
- 模板支持重命名、删除和复制模板。
- 项目更多菜单新增“复制项目”，可完整复制项目任务、父子层级、里程碑和依赖关系。
- 复制项目时会重新生成项目 id 和所有任务 id，并同步映射 `parentId` 与依赖任务 id，复制后自动切换到新项目。
- Sidebar 项目列表新增项目搜索框，支持按项目名称实时模糊搜索且不区分大小写。
- 项目搜索无匹配结果时显示“未找到项目”，并保留“+ 新建项目”入口。
- 右侧甘特图工具栏新增任务搜索框，支持按当前项目任务名称实时搜索。
- 新增任务筛选下拉，支持全部任务、普通任务、里程碑、父任务、全局关键任务和局部关键任务筛选。
- 搜索和筛选结果保留任务树层级：子任务命中时显示父级，父任务命中时显示其子任务。
- 甘特图与左侧任务列表同步应用搜索和筛选结果，清空搜索后恢复完整当前项目任务视图。

### Changed
- 里程碑规范为 0 工期项目节点：创建、编辑、拖拽和旧数据读取时都会强制 `start = end`。
- 里程碑通过状态不再由 `progress = 100` 自动推导，必须由用户手动确认；确认通过会记录通过时间并将进度设为 100。
- 里程碑延期改为派生 UI 状态，不写入 localStorage，避免节点状态颜色与关键路径颜色冲突。
- 里程碑状态使用徽标/按钮表达，任务条主体颜色仍保留给全局关键路径和局部关键路径样式。
- 任务列表状态列从“节点状态”调整为“状态”，普通任务显示计划状态，里程碑显示计划状态与节点确认状态的组合。
- 简化里程碑状态列展示，取消“已完成 · 未确认”等复合文案，里程碑仅显示确认态以避免内容挤压。
- 任务筛选新增已完成任务、已延期任务和待确认节点，和项目健康概览条联动。
- milestone 确认通过前会校验依赖是否满足，未满足时弹出风险提示，默认阻止，用户可选择强制通过。
- 状态列移除“依赖未满足 / 依赖异常”文字标签，仅保留主状态标签与告警图标，减少列内容拥挤。
- milestone 日期未到时的主状态从“未确认”改为“未开始”，降低节点确认语义的理解成本。
- 项目默认 activeProject 解析优先选择普通项目，避免模板在非主动点击时成为默认工作项目。
- `useTasks` 额外暴露完整任务行列表，用于搜索筛选时跨折叠层级匹配，同时保留原有展开/折叠视图逻辑。

### Fixed
- 修复项目重命名输入框每次输入后反复全选文本，导致中文输入法无法连续输入的问题。
- 修复计划开始日期未到但任务进度已大于 0 时仍显示“未开始”的状态判断问题，现在会显示为“进行中”。
- 修正项目健康概览统计口径：父任务不参与计数，普通任务状态与 milestone 节点确认状态分开统计；已延期只统计普通延期任务，待确认节点包含所有未通过且已到期/过期的节点。
- 修复 milestone “确认通过”在依赖未满足时被 hook 静默拦截、界面看起来无响应的问题；现在会按确认通过后的进度预判依赖风险并弹出确认框。

## [2026-04-26]
### Added
- 新增产品级 App Shell：全局 Header、固定 Sidebar 和右侧 Main 工作区布局。
- Sidebar 新增项目列表 UI，支持多项目展示、当前项目高亮和新建项目入口。
- 新增 `Project` 数据结构，项目包含 `id`、`name`、`tasks`、`createdAt`、`updatedAt`。
- 新增 `projectService`，统一管理项目读取、保存、创建、更新、删除与 activeProjectId 持久化。
- 支持从旧版单一 `tasks` localStorage 数据自动迁移为“默认项目”。
- Sidebar 支持项目切换、新建项目后自动切换、项目重命名和项目删除。
- 项目重命名支持更多菜单与双击项目名内联编辑，支持 Enter 保存、Esc 取消和失焦保存。
- 增加专业 CPM（Critical Path Method）关键路径计算，基于 Forward Pass / Backward Pass / Total Slack 识别关键任务。
- CPM 支持 FS、SS、FF 依赖类型与可选 lag。
- 关键路径任务自动标记 `isCritical`，普通任务和里程碑参与计算，父任务不参与 CPM 节点计算。
- 甘特图新增“关键路径”显示开关。
- 关键路径任务条和关键路径依赖箭头使用醒目红色样式展示。
- 检测到依赖循环时显示修复提示，并停止关键路径计算。
- 新增父任务内局部关键路径：点击父任务后，仅在该父任务的递归子任务范围内运行 CPM。
- 局部关键路径任务与依赖箭头使用橙色/虚线样式展示，且全局关键路径红色样式优先。
- 里程碑新增覆盖层视觉：以 16px 菱形和右侧任务名标签展示，支持普通、全局关键和局部关键状态。

### Changed
- localStorage 数据结构从单一任务列表升级为多项目列表，任务增删改、拖拽、依赖、关键路径和父任务汇总均限定在当前项目内。
- 主界面从居中卡片 Demo 布局调整为全屏工作台布局，Sidebar 固定宽度，Main 区域自适应填满剩余空间。
- 甘特图工作区改为占满 Main 可用空间，移除大卡片包裹和多余外边距，减弱边框与阴影。
- 顶部工具栏调整为纯操作区，视图切换、关键路径和新建任务按钮布局更紧凑一致。
- 优化甘特图日/周/月视图时间轴密度，固定每个时间单位宽度，避免随屏幕宽度无限拉伸。
- 为短项目补齐可视时间范围，让所有项目在日/周/月视图下都能稳定铺满甘特图工作区。
- 日视图表头改为星期与日期双行显示，窄列时保留日期展示。
- Main 工作区保留右侧视觉安全区，避免甘特图内容贴近屏幕边缘，同时不改变甘特图实际布局宽度。
- Sidebar 视觉层级、hover 状态、当前项目高亮和项目操作按钮显示时机进一步优化。
- 父任务关键状态由子任务派生显示，不参与 CPM 网络计算。
- CPM Forward Pass 使用当前甘特图任务开始日期作为最早开始下界，避免多个独立任务组被错误平移到同一起点。
- 依赖未显式设置 lag 时，CPM 不再从当前排期自动推导等待时间；任务间可见空档默认视为 slack。
- 点击页面空白区域会取消父任务选中状态，并隐藏局部关键路径。
- 局部关键路径任务样式从橙色填充改为橙色虚线边框，任务条本体保持原有颜色。
- 同时命中全局与局部关键路径时，保留红色全局填充并叠加橙色虚线局部标识；依赖线同理叠加偏移虚线。
- 缩小依赖箭头 marker 与线宽，降低箭头对任务条的视觉抢占。
- 里程碑局部关键状态改为橙色虚线外框，全局关键状态保持红色菱形。
- 提升甘特图任务浮窗层级，避免依赖箭头和里程碑覆盖层遮挡浮窗内容。
- 修复里程碑结束时间在任务列表中被第三方甘特库重置为开始时间的问题，列表和浮窗改用项目原始任务数据展示日期。
- 里程碑支持按起止时间显示可变长度长菱形，结束时间大于开始时间时不再只显示为一个点。
- 修正长菱形覆盖层与底层可拖拽时间条不重合的问题，让菱形左右边界跟随真实拖拽边界。
- 关键路径计算中的里程碑工期改为使用实际起止时间，保持与可变长度里程碑显示一致。
- localStorage 保存时会剥离依赖关系中的派生关键路径标记，保持旧数据兼容与存储干净。

### Fixed
- 修复 Sidebar 项目更多菜单被滚动容器裁剪，必须滚动才能看到“重命名/删除项目”的问题。
- 修复删除当前项目后 activeProjectId 可能失效的问题，删除后自动切换到剩余项目。
- 修复只有一个项目时仍可删除的问题，现在至少保留一个项目。
- 修复甘特图时间轴在大屏或短项目下宽度异常、右侧大面积空白或安全区被吃掉的问题。
- 修复日视图表头在第三方甘特图异步重绘后偶发退回单行日期的问题。
- 修复 Main Header 和甘特图工具栏重复展示项目名导致界面层级混乱的问题。
